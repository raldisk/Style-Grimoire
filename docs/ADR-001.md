# ADR-001: Deployment Architecture and API Interaction Strategy

**Status:** Accepted  
**Date:** 2026-03-08

---

## Context

Style Grimoire interacts with the Anthropic Messages API to generate style outputs.

The project currently runs inside the **Claude artifact runtime environment**, which provides:

- Managed authentication (auth headers injected automatically)
- Runtime API access (no `x-api-key` or `anthropic-version` headers required from client)
- Isolated execution environment

Because of this, several production concerns — API key management, proxying, networking — are handled automatically by the platform.

This ADR documents the reasoning behind the current approach and the migration path for standalone deployment.

---

## Decision: Current Implementation

The application communicates directly with the Anthropic API from the client runtime.

```
React UI
    │
    │  direct request (auth injected by runtime)
    ▼
Anthropic Messages API
```

**Reasons:**

- The Claude artifact runtime injects authentication automatically
- API access is sandboxed — no exposed credentials
- Simplified deployment and fast development iteration

---

## Alternative Architecture: Standalone Production Deployment

If deployed as a standalone web application, the recommended architecture is:

```
React UI
    │
    │  fetch("/api/generate")
    ▼
Backend Proxy (Node / Express)
    │
    │  authenticated request (x-api-key, anthropic-version)
    ▼
Anthropic Messages API
```

**Benefits:**

| Concern | Benefit |
|---------|---------|
| Security | Prevents API key exposure in browser bundle |
| Observability | Centralized logging and monitoring |
| Rate Limiting | Retry/backoff handled server-side |
| Cost Control | Prevents API abuse |

---

## Additional Enhancements (Future)

### Streaming Responses (SSE)

Switch from blocking responses to token streaming via Server-Sent Events.

```
Content-Type: text/event-stream

data: {"type":"content_block_delta","delta":{"text":"Hello"}}
data: {"type":"content_block_delta","delta":{"text":" world"}}
data: [DONE]
```

**Benefits:** Reduces perceived latency by 2–4 seconds on longer responses.

---

### Retry with Exponential Backoff

Already implemented client-side in `useStyleStudio.ts` via `retryFetch()`.

```
Retry up to 3 times on 429/529
Delays: 1s → 2s → 4s
```

For standalone deployment, this logic should move server-side.

---

### Runtime Schema Validation

Use Zod to validate:
- Incoming request payloads
- Streaming event structures
- API response shape

---

## Migration Path

If the project evolves into a production standalone application:

1. Introduce backend proxy (`/api/generate`)
2. Move API keys to server environment variables
3. Enable SSE token streaming
4. Move retry/backoff logic server-side
5. Add Zod runtime validation

These changes can be introduced incrementally without major refactoring.

---

## Consequences

**Positive:**
- Current architecture is simple and appropriate for the artifact runtime
- Minimal infrastructure required
- Fast development iteration

**Negative:**
- Directly depends on Claude artifact runtime features
- Not suitable as-is for standalone web hosting
